## Pattern Assignments

### `frontend/web/styles/globals.css` — Tailwind entry point

**Analog:** `frontend/_design-reference/styles.css` (naming reference only — do **not** copy CSS rules)

**What to mirror — font CSS-variable names** (`frontend/_design-reference/styles.css:35-36`):

```css
  --font-display: 'Manrope', system-ui, -apple-system, sans-serif;
  --font-body: 'Inter', system-ui, -apple-system, sans-serif;
```

**What Phase 1 actually writes** (Phase 2 — DSGN-01..06 — populates the `@theme` block; Phase 1 keeps it minimal):

```css
@import "tailwindcss";

/* Phase 2 (DSGN-01..06) populates @theme here, mirroring _design-reference/styles.css :root tokens. */
```

**Notes:**
- CONTEXT.md D-01 — Phase 1 must keep `globals.css` minimal: `@import "tailwindcss";` plus base reset only. **Do not** copy `_design-reference/styles.css` `:root` tokens into Phase 1 — that is Phase 2's work.
- CONTEXT.md `<canonical_refs>` and `<specifics>` — the **only** thing Phase 1 inherits from `_design-reference/styles.css` is the variable names `--font-display` and `--font-body` so Phase 2 can `@theme { --font-display: var(--font-display); ... }` without renaming.
- ROADMAP success criterion #4 — globals.css must be wired such that `next/font` exposes `--font-display` and `--font-body` (set on `<html>` by `app/layout.tsx`), not loaded from the Google Fonts CDN at runtime.
- CONTEXT.md `<decisions>` `Claude's Discretion` — exactly one `globals.css` is imported in `app/layout.tsx`. If `create-next-app` emits `app/globals.css`, planner picks: keep it OR move to `styles/globals.css` and remove the duplicate. FOUND-05 requires the `styles/` directory to exist regardless.

### `frontend/web/app/layout.tsx` — Root layout (loads fonts)

**Analog:** N/A — greenfield (no existing React code). The CSS-variable contract comes from `frontend/_design-reference/styles.css:35-36` (above).

**Convention to follow** (CONTEXT.md D-04 + `<decisions>` Discretion + ROADMAP success #4):

The root layout MUST:
1. Use `next/font/google` for both Manrope (display) and Inter (body). Subsets: `latin`. Weights: 400/500/600/700 minimum.
2. Pass `variable: '--font-display'` to the Manrope loader and `variable: '--font-body'` to the Inter loader (these names are **load-bearing** — they are the contract Phase 2's `@theme` block consumes; renaming them breaks the Phase 1 → Phase 2 handoff seam).
3. Apply both font CSS-variable classes on the `<html>` (or `<body>`) element so the variables are reachable from anywhere in the tree.
4. Import `globals.css` once.

**Anti-pattern to refuse** (`frontend/_design-reference/Recommend-a.html` — see CONTEXT.md `<code_context>` "Reusable Assets"):
- Pulling fonts via `<link rel="stylesheet" href="https://fonts.googleapis.com/...">`. CLAUDE.md and ROADMAP success #4 explicitly forbid the Google Fonts CDN at runtime. Use `next/font` only.

## Shared Patterns (cross-cutting conventions)

These are not patterns extracted from existing code (none exists). They are project-wide rules that the planner MUST surface in every relevant plan task.

### Convention 1 — "Frontend lives at `frontend/web/`"
**Source:** CLAUDE.md hard rule #1
**Apply to:** All files in this phase. Every new path begins with `frontend/web/`. The Pulumi backend (`__main__.py`, `functions/`) is read-only this milestone — do not edit, do not import, do not pattern-match against it.

### Convention 2 — "Never import JSX from `_design-reference/`"
**Source:** CLAUDE.md hard rule #2
**Apply to:** `app/layout.tsx`, `app/page.tsx`, `app/tokens/page.tsx`, and any future component file. The `_design-reference/` directory is for visual reference only. The single legitimate inheritance is the CSS-variable **names** `--font-display` / `--font-body` from `_design-reference/styles.css:35-36`, mirrored in `app/layout.tsx`'s `next/font` `variable:` options.

### Convention 3 — "All backend calls are mocked"
**Source:** CLAUDE.md hard rule #3
**Apply to:** `lib/api/.gitkeep` (or `index.ts`). Phase 1 creates the directory empty; do not add any code, types, or stubs here in Phase 1. Phase 4 fills it with the typed mock Cognito surface.

### Convention 4 — "Theme variables only" (Phase 2 onward)
**Source:** CLAUDE.md hard rule #4 + CONTEXT.md `<deferred>` (DSGN-06 lint rule deferred to Phase 2)
**Apply to:** Phase 1's placeholder pages should not litigate this — they may use minimal text without Tailwind utilities. But once Phase 2 ships, no hex colors and no px font-sizes inside `app/` or `components/`. Phase 1 lint config does **not** enforce this yet (DSGN-06 is Phase 2 territory).

### Convention 5 — "Branch naming: `feature/issue-{N}-{slug}`"
**Source:** CLAUDE.md hard rule #5
**Apply to:** Out-of-scope for file content but the planner should note that work happens on `feature/issue-90-foundation` (the active branch per `git status`).

### Convention 6 — "Tailwind v4 is CSS-first — no `tailwind.config.ts`"
**Source:** CONTEXT.md D-01
**Apply to:** No `tailwind.config.ts` is created. `globals.css` is the only place tokens live (and the `@theme` block stays empty until Phase 2).

### Convention 7 — Two routes only, exactly
**Source:** CONTEXT.md D-03 + ROADMAP success criteria #1 & #2
**Apply to:** `app/page.tsx` and `app/tokens/page.tsx`. No `app/_health/`, no `app/about/`, no `app/login/` placeholder. The verification step must exercise BOTH routes (`/` rendering AND a click-through navigation to `/tokens` succeeds), not just one (CONTEXT.md `<specifics>`).

### Convention 8 — `components/` stays empty
**Source:** CONTEXT.md D-02
**Apply to:** Do not let `shadcn init` scaffold a demo component (e.g. `button.tsx`). If the current shadcn CLI auto-adds one, remove it. Phase 3 owns the `components/` surface.

### Convention 9 — Verifier must grep the built output for `fonts.googleapis.com`
**Source:** ROADMAP success criterion #4 + CONTEXT.md `<specifics>`
**Apply to:** The verification plan must include a step that runs `pnpm build` and greps the produced `.next/` (or static HTML) output for `fonts.googleapis.com` — expecting **zero hits**. This is how "no Google Fonts CDN at runtime" is proven, not assumed.
